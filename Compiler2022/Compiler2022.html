Complier 2022

This is an example of the compiler I developed in 2022 as is from the state it was in when presented to clients.

Goal: compile SQL like statements into JSON for processing.



Requirements:


- Allow the creation and use of custom metrics
- preform lexical, syntax and semantic analysis
- use python and ReGex
- give each a UUID and a readable name
- Ensure vairables don't overlap
- Compile to a JSON structure for further processing indicating operator and input

My implementation:


- use a dictionary to allow additional metrics.
- append the strings to the TOKENS list to reserve the words
- Use ReGex to compress and evaluate statements.
- Generate a UUID using uuid library and place in the JSON structure
- Add a counter which adds on to the end of every created metric to ensure no overlap.
- During evaluation create a JSON struct

Final Result:

A basic one line compiler with the following structure.
1. input: statement
2. input: name
4. output: list of metrics
3. output: errors, or the metric and its JSON structure.
4. Debug output: the full JSON structure created and stored.
